What is a “Group”? Do I need to use it?

Yes — you should use Groups. A Group is simply a job role attached to a login account. The system has three groups:

  • Sales — sales staff
  • Accounting — finance / accountants
  • Admin — manager or owner with full control

When you create a user, you pick one Group. That decides what they can see in the ERP (e.g. Sales cannot see supplier costs on invoices).

Groups are not optional if you want different access for sales vs accounting. Without a group, the user may not behave as expected.

In Django Admin, Groups appear under Authentication and Authorization → Groups. You normally do not create new groups — only assign users to Sales, Accounting, or Admin (already set up on your server).

Two different things (do not confuse them)

Login User + Group Employee (staff list)
Where Admin → Users → Groups Admin → Employees
Purpose Username, password, and permissions Name on invoices and salesman reports
Example “Rana” logs in as Sales “Rana” appears as salesperson on an invoice

The Employee role (Sales / Accounting / Admin on the employee record) is for reporting labels, not for login security. Login security comes from the user’s Group.

Who should access the Admin panel (/admin/)?

Person Group Staff? Superuser?
Developer / IT support — Yes Yes (you only)
CEO / owner Admin Yes No (unless needed)
Main accountant Accounting Optional No
Other accountants Accounting No No
Sales employees Sales No No

Staff = checkbox on the user page; required to open /admin/.
Superuser = full technical control; keep for 1–2 people only.

“Is main accountant” — should I use it?

Yes — for one person only: your lead / head accountant.

When adding or editing a user in Admin, under Profile, tick Is main accountant for that person only.

Effect: New invoices automatically default the salesperson/employee field to that person’s Employee record (if linked).

Also for the main accountant:

  1. Admin → Employees → create or edit their record (name, role Accounting).
  2. Link User = their login account.
  3. User → Group = Accounting.
  4. Tick Is main accountant on their profile.

Other accountants: Group Accounting, do not tick main accountant.

Step-by-step: create a new sales employee

  1. Admin → Employees → Add → name, role Sales, active.
  2. Admin → Users → Add user → username & password.
  3. On user: tick Staff = off. Superuser = off.
  4. Under Groups, choose Sales only → Save.
  5. Edit the Employee record → link User to this account.
  6. Send them the main site URL and login (not /admin/).

Step-by-step: create the main accountant

  1. Admin → Users → Add user → username & password.
  2. Group = Accounting. Staff = optional. Superuser = off.
  3. Tick Is main accountant on profile → Save.
  4. Admin → Employees → add their name, role Accounting, link User.

Step-by-step: CEO / owner

  1. User with Group Admin, Staff ticked.
  2. Uses main ERP for overview and /admin/ to manage users if needed.
  3. Do not give every employee superuser access.

Security reminder

  • Give Sales the Sales group — not Admin.
  • Keep superuser for technical support only.
  • Change passwords if they were shared insecurely.
